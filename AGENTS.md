# Repository Rules

## Git Workflow

- `master` is the protected branch. Do not commit directly to `master`.
- All work must start from a short-lived branch created from `master`.
- Changes must be merged back into `master` only through a pull request (PR) after review.
- Keep branches focused on a single task or feature.
- Before opening a PR, make sure the branch is up to date with `master` and that local checks pass when available.

## Branch Naming

- Use descriptive names such as `feature/add-users-route`, `fix/health-endpoint`, or `chore/setup-eslint`.
